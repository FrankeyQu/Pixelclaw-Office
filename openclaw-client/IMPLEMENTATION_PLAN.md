# 功能实施计划 - 6 项改进

## 设计原则
- **无功能冲突**：任务/Quest 合并为统一任务面板；数据面板独立
- **风格融合**：沿用现有 `.control-group`、`btn`、`stat-card` 样式
- **质量保证**：分阶段实施，每阶段可独立验证
- **决策保留**：复杂选项在代码中标注 `// TODO: 用户决策`

## 实施顺序

| 阶段 | 功能 | 改动范围 | 风险 |
|------|------|----------|------|
| 1 | 状态可视化 (error/waiting) | pixel-office.js + index.html | 低 |
| 2 | 面板 Tab 化 | index.html 结构 + 少量 JS | 低 |
| 3 | 任务面板 + Quest Log | 新模块 + index.html | 中 |
| 4 | 数据面板 (成本 Mock) | 新区块 + index.html | 低 |
| 5 | 地图编辑增强 | map-editor.js | 中 |
| 6 | 组件化预留 | 保持 Tab 结构，后续可扩展 | - |

## 功能冲突规避
- **任务 vs Quest Log**：合并为「任务」Tab，含列：待办 | 进行中 | 需确认 | 完成
- **Agent 列表**：保留在「总览」Tab，任务可关联 agentId 点击定位
- **成本数据**：独立「数据」Tab，明确标注为示例数据
