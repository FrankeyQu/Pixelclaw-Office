# Pixelclaw Office 付费元素与许可证设计

> **实现说明**：已实现「一条复杂码」方案，见 `feature/license-key` 分支。用户购买后获得一条 PXO- 开头的码，粘贴即用，完全离线。

## 一、业务模型

### 1.1 售卖范围
- **核心功能**：免费，始终可用
- **可选元素**：付费解锁，如：
  - 猫咪、宠物系列
  - 布告栏、白板
  - 特色装饰、主题包
  - 高级人物模型

### 1.2 核心矛盾
- 用户多部署在**内网**，无法实时联网验签
- 需在**离线可用**的前提下，兼顾防滥用与实现成本

---

## 二、设计原则

1. **离线优先**：许可证校验不依赖实时网络
2. **合规优先**：面向企业/内网用户，主要靠合规而非技术强约束
3. **实现简单**：避免复杂 DRM，降低维护成本
4. **体验友好**：未购买元素不显示/不可选，不弹窗骚扰

---

## 三、许可证方案对比

| 方案 | 内网可用 | 防共享 | 实现难度 | 适用场景 |
|------|----------|--------|----------|----------|
| 在线实时验签 | ❌ | 高 | 低 | 排除 |
| 离线 .lic 文件 + 机器码 | ✅ | 中高 | 中 | 个人/小团队 |
| 离线 .lic 文件（无机器码） | ✅ | 中 | 低 | 企业站点授权 |
| 密钥 + 离线激活 | ✅ | 中高 | 高 | 大型软件 |

**推荐**：以「离线 .lic 文件」为主，按客户类型选择是否绑定机器码。

---

## 四、推荐架构

### 4.1 许可证文件（.lic）

```
pixeloffice.lic   # 放置在应用同目录或指定路径
```

**格式（JSON，签名保护）**：

```json
{
  "v": 1,
  "products": ["cat-pack", "bulletin-board", "theme-premium"],
  "holder": "某公司 / 个人名",
  "type": "site",
  "expiry": "2026-12-31",
  "issuedAt": "2025-03-14",
  "sig": "base64-rsa-signature-of-above"
}
```

| 字段 | 说明 |
|------|------|
| products | 已授权产品 ID 列表 |
| type | `personal`=单机绑定, `site`=站点/企业不绑机器 |
| expiry | 到期日，空表示永久 |
| sig | 对 payload 的 RSA 签名，用你私钥生成 |

### 4.2 授权类型

| 类型 | 机器码 | 适用 | 说明 |
|------|--------|------|------|
| personal | 绑定 | 个人/小团队 | 一个 .lic 仅在本机有效 |
| site | 不绑定 | 企业/内网 | 一个 .lic 全公司可用 |

**内网部署**：优先卖 `site` 授权，不校验机器码，完全离线。

## 五、技术实现

### 5.1 签名与验签

```
你持有：私钥（保密，仅在签发时用）
应用内置：公钥（可公开，用于验签）
```

- 用户购买后，你在后台用私钥对 license payload 签名，生成 .lic
- 运行时用内置公钥验证 `sig`，防止篡改和伪造

### 5.2 机器码（仅 type=personal 时使用）

```
指纹 = hash(MAC + CPU 核心数 +  hostname 等)
```

- 浏览器环境可用的稳定信息有限，可考虑：
  - `navigator.hardwareConcurrency`
  - `screen.width/height`
  - `localStorage` 随机 ID（首次生成后持久化）
- 纯前端无法拿到真实 MAC/CPU，可接受一定误差，重点增加共享成本

### 5.3 运行时流程

```
启动 / 切换场景
  → 读取 .lic（同目录 或 配置的路径）
  → 验签
  → 若 type=personal，校验机器码
  → 检查 expiry
  → 得到 products 列表
  → 启用对应付费元素，其余保持隐藏/禁用
```

### 5.4 元素与产品映射

```javascript
// 产品 ID 与资源的映射
const PRODUCT_ELEMENTS = {
  'cat-pack': ['cat-orange', 'cat-gray', 'cat-black'],
  'bulletin-board': ['bulletin-board'],
  'theme-premium': ['deco-premium-1', 'deco-premium-2']
};

// 运行时
function hasProduct(productId) {
  return license && license.products.includes(productId);
}

function canUseAsset(assetKey) {
  const product = getProductForAsset(assetKey);
  return !product || hasProduct(product);
}
```

## 六、内网部署专用设计

### 6.1 购买与交付流程

```
1. 用户在你的官网/店铺购买
2. 填写：公司名、授权类型（site/personal）、数量
3. 你生成 .lic 文件
4. 通过邮件/网盘交付 .lic
5. 用户将 .lic 放到部署目录，或通过「许可证管理」页面上传
```

### 6.2 .lic 放置方式

| 方式 | 适用 | 说明 |
|------|------|------|
| 同目录 | 桌面/本地部署 | `index.html` 同目录放 `pixeloffice.lic` |
| 上传 | 纯前端部署 | 在管理页上传 .lic，存入 localStorage（注意清缓存会失效） |
| 配置 URL | 有内网文件服务 | 如 `licenseUrl: "/config/pixeloffice.lic"`，应用启动时 fetch |

**内网建议**：支持「同目录」+「配置 URL」，便于挂载到内网文件服务器。

### 6.3 无网络时的完整流程

- 购买：用户在外网完成支付
- 交付：邮件发送 .lic
- 使用：将 .lic 拷入内网部署目录，无需联网即可使用

## 七、安全措施（适度即可）

| 措施 | 作用 | 成本 |
|------|------|------|
| RSA 验签 | 防止伪造、篡改 .lic | 低 |
| 代码混淆 | 提高逆向难度 | 低 |
| 机器码（personal） | 限制单机授权共享 | 中 |
| 公钥拆分/编码 | 略微增加分析难度 | 低 |
| 水印/埋点 | 泄露追溯（可选） | 中 |

**不建议**：复杂 DRM、在线心跳、强反调试——成本高且与内网场景冲突。

|------|------|------|

| 7 | 许可证管理 UI | 上传、状态展示 |

## 十、定价与授权建议

- **个人**：按产品包定价，绑定单机
- **企业/内网**：站点授权，按人数或一次性买断，不绑机器
- **试用**：可提供 7 天全功能试用 .lic，便于内网评估

---

## 十一、总结

| 问题 | 方案 |
|------|------|
| 内网无网 | 离线 .lic，不依赖实时联网 |
| 防伪造 | RSA 签名 + 公钥验签 |
| 防共享 | 企业用 site 授权；个人用 personal + 机器码 |
| 交付 | 邮件/网盘发 .lic，用户放入部署目录或通过配置 URL 加载 |

整体策略：**离线可用 + 签名防伪 + 合规优先**，在保证内网可用和实现简单的前提下，合理控制滥用。
