# 我的资产 - 功能规划方案

> **实现说明**：已实现于 `feature/my-assets` 分支。入口「我的资产」→ `my-assets.html`。

## 一、需求摘要

| 序号 | 需求 | 说明 |
|------|------|------|
| 1 | 入口改造 | 将「许可证」控制组改为「我的资产」，点击进入独立页面 |
| 2 | 兑换码兑换 | 支持输入兑换码/许可证码激活 |
| 3 | 新元素预览 | 展示付费元素（如猫咪、布告栏等）的预览与解锁状态 |
| 4 | 资产总览 | 展示已购产品、剩余时间等信息 |

---

## 二、整体架构

### 2.1 入口设计

**位置**：`index.html` 总览 Tab，原「许可证」控制组（第 4 个 control-group）

**改造**：
- 标题：许可证 → **我的资产**
- 内容：精简为「摘要卡片」+「进入」按钮
  - 未激活：显示「未激活」+ 按钮「进入我的资产」
  - 已激活：显示已购数量、到期提示（如「剩余 280 天」或「永久」）+ 按钮「进入我的资产」

**跳转**：点击按钮 → 打开 `my-assets.html`（新页面）

### 2.2 新页面：my-assets.html

独立页面，包含以下区块（自上而下）：

```
┌─────────────────────────────────────┐
│  我的资产                            │
├─────────────────────────────────────┤
│  【资产总览】                         │
│  - 授权状态：已激活 / 未激活           │
│  - 已购产品：cat-pack、bulletin-board │
│  - 到期时间：2026-12-31 或 永久       │
│  - 剩余天数：280 天（若有期限）        │
├─────────────────────────────────────┤
│  【兑换码兑换】                       │
│  - 输入框（粘贴许可证码/兑换码）        │
│  - 兑换按钮                          │
├─────────────────────────────────────┤
│  【新元素预览】                       │
│  - 网格展示各付费元素                  │
│  - 每项：缩略图、名称、状态(已解锁/未解锁)│
│  - 未解锁项可显示「购买解锁」占位（可后续接购买链接）│
├─────────────────────────────────────┤
│  [返回主页面]                         │
└─────────────────────────────────────┘
```

## 三、模块划分

### 3.1 文件结构

```
openclaw-client/
├── index.html              # 修改：许可证 → 我的资产入口
├── my-assets.html          # 新增：我的资产独立页面
├── my-assets.js            # 新增：我的资产页面逻辑
├── license.js              # 复用：激活、验签、缓存
├── pixel-office.js         # 修改：updateLicenseUI 适配新入口
└── assets/                 # 元素预览图（可选，可先用占位或文字）
    └── previews/
        ├── cat-pack.png
        └── bulletin-board.png
```

### 3.2 数据流

- **许可证数据**：沿用 `License.getCached()`、`License.activate()`、`License.clear()`
- **剩余天数**：由 `expiry` 计算 `Math.ceil((new Date(expiry) - new Date()) / 86400000)`
- **元素与产品映射**：沿用 `PRODUCT_ELEMENTS` 或集中配置

## 四、功能详情

### 4.1 资产总览

| 字段 | 来源 | 展示逻辑 |
|------|------|----------|
| 授权状态 | `License.getCached()` | 有缓存=已激活，无=未激活 |
| 已购产品 | `cached.products` | 列表展示，可配中文名映射 |
| 到期时间 | `cached.expiry` | 有则显示日期，无则显示「永久」 |
| 剩余天数 | `expiry` 与当前日期差 | 仅有时效授权显示 |

### 4.2 兑换码兑换

- 与现有「许可证激活」同一逻辑：输入 PXO- 开头的码，调用 `License.activate(key)`
- 成功后刷新本页总览与预览状态
- 可保留「兑换码」命名，便于与「许可证码」区分展示，底层仍为同一格式

### 4.3 新元素预览

- **产品列表**：从配置读取，如 `['cat-pack', 'bulletin-board']`
- **每项展示**：
  - 名称（可配置中文）
  - 预览图（有则用图，无则占位/图标）
  - 状态：`License.hasProduct(id)` → 已解锁 / 未解锁
- **扩展**：未解锁项可预留「购买」按钮，后续对接购买页

### 4.4 产品配置（集中管理）

```javascript
// 可放在 license.js 或 my-assets.js
const PRODUCTS = [
  { id: 'cat-pack', name: '猫咪系列', preview: 'assets/previews/cat-pack.png' },
  { id: 'bulletin-board', name: '布告栏', preview: 'assets/previews/bulletin-board.png' }
];
```

## 七、待确认点

1. **入口形式**：独立页面 `my-assets.html` 是否合适，还是希望用弹窗/侧滑 panel 嵌在 index 内？
2. **兑换码 vs 许可证码**：是否保持同一 PXO- 格式，仅文案区分？或需要单独的「短兑换码」格式？
3. **预览图**：是否已有设计稿/素材，还是先用占位图或纯文字？
4. **分支基准**：从 `feature/license-key` 还是 `main` 拉新分支？（建议 `feature/license-key`，因依赖许可证逻辑）
